# 模块化对战AI架构设计

## 架构概述

本文档描述了一个模块化的宝可梦对战AI架构设计，包含五个核心模块，每个模块负责特定的功能，相互协作完成对战决策过程。

## 模块设计

### 1. 游戏规则模块

**功能**：内置宝可梦对战的完整（或部分，根据实际工作量决定）游戏规则，用于模拟对战行为。

**核心职责**：
- 实现宝可梦对战的基本规则和机制
- 处理状态异常、天气、场地效果等特殊规则
- 模拟招式使用、伤害计算、属性克制等核心战斗逻辑
- 提供规则验证和合法性检查

**实现要点**：
- 基于官方宝可梦游戏规则实现
- 支持不同世代的规则差异
- 提供规则查询和模拟接口
- 与推理计算模块紧密集成

### 2. 信息获取模块

**功能**：获取对战中明确显示的公开信息（如战报信息、场地信息等）。

**核心职责**：
- 解析对战界面中的战报信息
- 获取场地状态（天气、地形、陷阱等）
- 收集双方宝可梦的可见信息（等级、性别、特性激活等）
- 监控对战过程中的状态变化

**实现要点**：
- 利用 `battle.data` 对象获取对战数据
- 解析回调函数中的参数获取实时信息
- 建立信息缓存机制，减少重复查询
- 提供标准化的信息访问接口

### 3. 信息推测模块

**功能**：基于已有信息对未知信息（如对手的可能配置，速度线，攻击与耐久等）进行推测。

**核心职责**：
- 推测对手宝可梦的可能配置（技能、特性、道具）
- 分析对手的速度线和可能的努力值分配
- 评估对手的攻击和耐久水平
- 预测对手的可能战术和行动

**实现要点**：
- 基于对战历史数据进行统计分析
- 利用常见配置数据库进行匹配
- 实现概率模型评估不同可能性
- 结合对手的行动模式进行推断

### 4. 推理计算模块

**功能**：计算行动的伤害分布（基于命中率、CT概率、伤害随机数等）。

**核心职责**：
- 计算招式的伤害范围和分布
- 考虑命中率、暴击率等随机因素
- 评估不同行动的风险和收益
- 模拟不同行动的可能结果

**实现要点**：
- 实现精确的伤害计算公式
- 考虑各种修正因素（属性克制、STAB、天气等）
- 生成伤害分布的概率模型
- 提供多种场景的模拟计算

### 5. 行动决策模块

**功能**：建立行动对于对战胜率影响和收益的评价体系。

**核心职责**：
- 评估不同行动的对战价值
- 建立基于胜率的评价函数
- 选择最优行动策略
- 考虑长期和短期收益的平衡

**实现要点**：
- 实现基于博弈论的决策模型
- 考虑不同对战阶段的策略调整
- 结合队伍整体战术进行决策
- 提供决策理由和解释

## 模块间协作

### 数据流

1. **信息获取模块** → **信息推测模块**：提供基础信息用于推测
2. **信息获取模块** + **信息推测模块** → **推理计算模块**：提供计算所需的参数
3. **推理计算模块** + **游戏规则模块** → **行动决策模块**：提供决策依据
4. **行动决策模块** → 执行行动

### 依赖关系

- **游戏规则模块**：被其他所有模块依赖
- **信息获取模块**：被信息推测和推理计算模块依赖
- **信息推测模块**：被推理计算模块依赖
- **推理计算模块**：被行动决策模块依赖
- **行动决策模块**：依赖其他所有模块

## 实现建议

1. **模块化设计**：每个模块独立实现，通过明确的接口进行交互
2. **可扩展性**：设计时考虑未来功能的扩展，如支持不同世代的规则
3. **性能优化**：对于计算密集型任务（如伤害计算）进行优化
4. **可测试性**：每个模块都应该可以独立测试
5. **文档完善**：为每个模块提供详细的文档和使用说明

## 技术栈

- **语言**：JavaScript（与现有代码保持一致）
- **数据存储**：JSON文件存储配置和数据
- **计算库**：使用内置数学函数进行概率计算
- **测试框架**：使用简单的断言测试验证模块功能

## 与现有代码的关系

- **20201227.js**：作为现有代码的参考，提取核心逻辑到新的模块中
- **sys对象**：用于获取宝可梦数据和系统功能
- **battle对象**：用于获取对战信息和执行对战操作
- **回调函数**：用于接收对战事件和状态变化
