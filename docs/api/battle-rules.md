# 宝可梦对战规则与机制参考

本文件详细说明宝可梦战斗AI系统中涉及的对战规则、状态异常、伤害计算等相关机制。

## 对战规则（Clause）

| 编号 | 规则名称 | 说明 |
|------|---------|------|
| 0 | Sleep Clause | 不允许在对方已经有1只宝可梦睡眠时再催眠一只宝可梦 |
| 1 | Freeze Clause | 不允许在对方已经有1只宝可梦冰冻时再冰冻一只宝可梦 |
| 2 | Disallow Specs | 不允许观战 |
| 3 | Item Clause | 不允许同队中的多只宝可梦携带同种道具 |
| 4 | Challenge Cup | 对战中双方宝可梦被替换成接近随机的阵容玩法 |
| 5 | No Time Out | 对战不限时间 |
| 6 | Species Clause | 同队不允许出现多只全国编号相同的宝可梦 |
| 7 | Team Preview | 对战开始前可以预览双方的宝可梦，并调整首发 |
| 8 | Self KO Clause | 对战中如果使用招式导致双方在这次攻击中同时倒下，则使用招式的一方判负 |
| 9 | Inverted Battle | 对战中的属性克制关系反转 |

## 状态异常（Status）

| 编号 | 状态名称 | 说明 |
|------|---------|------|
| 0 | 正常 | 无异常状态 |
| 1 | 烧伤 | 每回合结束时损失1/8最大HP，物理攻击威力减半 |
| 2 | 中毒 | 每回合结束时损失1/8最大HP |
| 3 | 麻痹 | 50%概率无法行动，速度变为原来的1/2 |
| 4 | 冰冻 | 无法行动，受到火属性攻击时解除 |
| 5 | 睡眠 | 无法行动，持续1-3回合 |
| 6 | 混乱 | 33%概率攻击自己，持续1-4回合 |

## 伤害计算相关

### 效果倍率（Effectiveness）

| 数值 | 效果 | 说明 |
|------|------|------|
| 0 | 无效 | 技能对目标完全无效 |
| 1 | 效果很差 | 伤害为正常的1/4 |
| 2 | 效果不好 | 伤害为正常的1/2 |
| 4 | 正常 | 伤害为正常的1倍 |
| 8 | 效果很好 | 伤害为正常的2倍 |
| 16 | 效果极佳 | 伤害为正常的4倍 |

### 伤害值表示

- **我方宝可梦**：`onDamageDone` 中的 `damage` 参数表示损失的HP数值
- **对方宝可梦**：`onDamageDone` 中的 `damage` 参数表示损失的HP百分比

## 队伍与位置

- 每个队伍最多包含6只宝可梦，索引为0-5
- 索引0通常表示当前场上的宝可梦
- 索引1-5表示场下的宝可梦
- `onSendOut` 中的 `prevIndex` 参数表示宝可梦从队伍中的哪个位置派出

## 消息编号系统

在回调函数中，以下参数不是实际的编号，而是消息类型编号：

- `onItemMessage` 中的 `item` 参数：道具消息编号
- `onMoveMessage` 中的 `move` 参数：技能消息编号
- `onAbilityMessage` 中的 `ab` 参数：特性消息编号

这些消息编号需要通过分析来确定具体的道具/技能/特性。

## 对战流程

1. **对战开始**：双方展示队伍，选择首发宝可梦
2. **回合进行**：
   - 双方宝可梦行动（根据速度决定先后顺序）
   - 使用技能、道具等
   - 处理伤害、状态变化等
3. **宝可梦交换**：当宝可梦被击倒或训练家主动交换时
4. **对战结束**：当一方所有宝可梦被击倒，或一方认输，或达成平局

## 特殊机制

- **会心一击**：伤害提升50%，忽略对方的防御强化
- **畏缩**：宝可梦无法行动一回合
- **Z招式**：需要持有对应的Z纯晶，威力大幅提升
- **超级进化**：需要持有超级石，能力和特性可能发生变化

## 注意事项

- 对战规则会影响对战的进行方式，需要在AI决策中考虑
- 状态异常会影响宝可梦的行动能力和战斗表现
- 伤害计算需要考虑属性克制、等级、个体值、努力值等多种因素
- 消息编号系统需要通过分析来确定具体的游戏元素
