- [ ] 同一对局中，每只候选的 `standard:N` 日志值与 v1.1.2 完全一致（抽取未改变任何评分）
- [ ] `好的防守选择`（getGoodForSwitch 返回）列表与 v1.1.2 一致
- [ ] 单属性宝可梦（type2=18）的 standard 计算正常，未因抽取引入 type2 误判

## B. 改动 F：standard 主导换人目标

- [ ] **case6-R7 复盘**：换人候选含西狮海壬(standard 高)与三首恶龙(standard 负、被克制)时，实际换上西狮海壬，不再随机抽中被克制的三首恶龙
- [ ] **case4-R6 复盘**：最优候选(银伴战兽 standard 低于 50 进不了攻击列表)时，兜底按全体 standard 选中它，而非换上被克制的音波龙
- [ ] 换人目标的 standard 显著为负(≤ -50 被克制)时，只要有更优候选就不会被选中
- [ ] 多个候选 standard 接近时仍保留随机性（不是每次都选同一只）
- [ ] passive 换人（KO 后被迫换）也走 standard 选最优

## C. 改动 E：被秒风险驱动换人

**getFoeThreatToMe 输出验证**：

- [ ] 日志 `getFoeThreatToMe threat:N bestDmg:X spd:AvsB` 正常输出
- [ ] `bestDmg` 落在合理区间（满血被一击秒时应 ≥1.0，普通招约 0.3~0.8）
- [ ] 对手速度更快时 `spd:` 显示我方速度 < 对手速度区间，threat 含速度加成
- [ ] 对手未亮相任何攻击招时，bestDmg 用本系假设（威力80）估算，不为 0

**换人触发验证**：

- [ ] **case7-R3 复盘**：火焰鸟满血对捷拉奥拉，threat ≥55 且场下有河马兽(good)，触发换人换走火焰鸟（不再被秒）
- [ ] **case7-R7 复盘**：河马兽对妙蛙花，场下有钢铠鸦，高威胁时触发换人
- [ ] 高威胁但**场下无合适候选**（三档列表全空）时，**不触发换人**（验证硬性前提，避免换人白送）
- [ ] 面对对手 setup sweeper 时，未因威胁高反复无意义换人送钉子/送人头

## D. 不退化回归（重点：历史包袱是过度换人）

> AI 接管前一直在调过度换人问题，v1.2 让换人更依赖 standard + 新增换人维度，须重点确认未过度换人。

- [ ] 跑若干完整对局，换人总频率与 v1.1.2 量级相当（仅目标更优 + 被秒前会换，不是每回合都换）
- [ ] 未出现"满血健康宝可梦无威胁却频繁换人"
- [ ] 未出现"换上→换下→再换上"的来回拉锯
- [ ] 若发现过度换人：优先回退改动 F 的兜底分支（`pickBySwitchStandard(switchesList,[])`），再考虑调高被秒维度阈值

## D2. 改动 G：确定性决策不被 softmax 覆盖

- [ ] 打印"先制斩杀生效"/"斩杀技能生效"等确定性决策日志的回合，**不再**跟随一行 `softmax:` 输出（说明 softmax 被跳过）
- [ ] **case3-2 复盘**：象牙猪类残局对 +1 高攻对手，冰砾（先制）"先制斩杀生效"后，实际打出的就是冰砾，而非被纯伤害更高的非先制招覆盖
- [ ] 变化招（ret≥3）决定使用后，实际打出该变化招，不被 softmax 用攻击招覆盖
- [ ] 普通攻击招（无特殊机制）回合仍正常输出 `softmax:` 行并随机化（确认普通招出口未被误标记）

## D3. 改动 H：残局收敛唯一解

- [ ] 对手只剩最后一只时，日志 `thresh=0.97`，`candidates=1`（除非有真等价同分招）
