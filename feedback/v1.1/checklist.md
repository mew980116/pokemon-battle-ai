## A. 机制验证（每项需至少触发 1 次并确认行为正确）

### A1. getOpponentDecisionPool（继承 v1.0）

- [ ] 开局第 1 回合 pool 在 70-80 之间（对手全未知）
- [ ] 对手被 KO 2 只后 pool 明显下降
- [ ] 对手使用剑舞/蝶舞后，下一回合 pool 下降约 25 分
- [ ] pool 值始终在 [0, 100] 范围内

**验证方式**：在日志中找 `softmax:` 行，观察 `pool=` 字段。

### A2. 自损招式惩罚（改动 B，继承 v1.0）

- [ ] 流星群进场首回合（pool 高）出现概率明显低于其他 STAB 招式
- [ ] 流星群能 OHKO 时：AI 正常使用，无惩罚
- [ ] 近身战类惩罚系数比流星群更轻，偶尔被选中

### A3. Path 1 强化换人（改动 A，继承 v1.0）

- [ ] 对手 +2 强化，场下有 standard > 100 候选时，换人率明显提升
- [ ] 我方能 OHKO（maxDamagePercent ≥ 0.9）时不换人

---

### A4. 温度 T 随局面变化（改动 D v2 核心）

- [ ] 对手攻击强化 = +2 时，T ≈ 1.2（2.0 × 0.60）——日志 `T=` 字段确认
- [ ] 对手攻击强化 = +3 时，T ≈ 0.9（2.0 × 0.45）
- [ ] 我方 HP < 25% 时，T 额外下降（× 0.55）：低血局 T 应 ≤ 0.7 左右
- [ ] 双方均衡、switchP > 65 时，T 应接近 2.5（2.0 × 1.25）
- [ ] T 始终在 [0.4, 4.0] 范围内，未见越界

**验证方式**：记录同一宝可梦在不同局面下的 `T=` 值，对比差异。

### A5. softmax 每回合触发（v1.1 最关键变化）

- [ ] **普通出招回合**（非 KO、非换人）：`softmax:` 行始终出现（candidates ≥ 1）
- [ ] 含属性克制的场景（AI 有 2× 以上克制招）：candidates ≥ 2，弱招出现在候选列表
- [ ] 同一局面重复触发，出现不同招式选择（随机性确认）
- [ ] 单招式可用时：candidates=1，不改变结果

**预期对比**：v1.0 中 40 回合仅触发 3 次 softmax；v1.1 应接近 40/40。

### A6. 权重分布合理性

对比 T 不同时的权重分布：

- [ ] T≈2.0，movepow 比 2:1 时：强招权重约 100，弱招约 70（概率约 59% vs 41%）
- [ ] T≈0.5，movepow 比 2:1 时：强招权重约 100，弱招约 17（概率约 85% vs 15%）
- [ ] T≈4.0，movepow 比 2:1 时：强招权重约 100，弱招约 84（概率约 54% vs 46%）
- [ ] 权重归一化后最大值始终为 100，最小值 ≥ 1

**验证方式**：从 `[招式列表]=[权重列表]` 字段手动验算 `movepow^(1/T)` 比值。

### A7. estimateFoeSwitchProb 行为

- [ ] 对手 bench 全未亮相（pokeNum=0）：switchP=0，bench=0，日志确认
- [ ] 对手有已知 counter（如草属性 bench 对我方水系）：switchP 应 ≥ 50
- [ ] 对手有已知 check 但有岩钉伤害（如飞行系 bench，我方岩钉已布）：switchP 应低于无钉时
- [ ] 对手所有 bench 均已 KO：switchP=0（无可换宝可梦）
- [ ] switchP 始终在 [0, 100] 范围内

**验证字段**：`softmax:` 行的 `switchP=` 和 `bench=`。

### A8. UT/VS 加成效果

- [ ] switchP > 60 且 AI 持有 UT/VS 时：`[UT/VS 名]=` 权重应接近最大或明显高于同 movepow 的其他招
- [ ] switchP = 0 时：UT/VS 权重应与同 movepow 的普通招式无差异
- [ ] switchP=80 时，UT（movepow≈70）与 movepow=100 的普通招竞争：UT 加成后约 70 × 1.64 = 115，应高于 100

**验证方式**：观察对手有已知 counter 时 AI 的 UT/VS 使用频率。

### A9. 岩钉战略分

- [ ] 我方持有岩钉（446），对手 bench 岩钉未布时：岩钉出现在候选列表，stratPow=70
- [ ] 典型 maxmovepow 约 200-300 时，T=2.0：岩钉权重约 100^(1/2) = 8.4，最高招约 17.3 → 岩钉约占 33%
- [ ] 岩钉已布后：岩钉不再出现在候选列表
- [ ] 地钉未满（spikesLevel < 3）且岩钉已布：地钉 stratPow=40，出现在候选但权重低于主攻招

**注意**：岩钉 stratPow=70 与 movepow=200+ 的招式竞争比 T=2 时约为 8.4:(14+)，约占 37% 以下。如出现下岩钉过于频繁则需上调 maxmovepow 权重或下调 stratPow。

### A10. Bench HP 与入场伤害因子（待实测确认 API 行为）

- [ ] **`hpPercentageWhenLeave` 初始值**：对手 bench pokemon 从未出场时，该字段值是多少？
  - [ ] = 0（代码将其视为满血 1.0）→ 逻辑正确
  - [ ] = 其他值（如 1.0）→ 逻辑同样正确
  - [ ] = -1 或异常值 → 需 guard，switchWill 可能算出负数
- [ ] **`sys.pokeType2` 单属性宝可梦**：返回值是否为 18？
  - [ ] = 18 → `bt2 < 18` 判断正确，不调用 `typechart(msType, 18)`
  - [ ] ≠ 18（如返回 type1 或 0）→ 需确认 typechart 对该值的处理

**验证方式**：在 `test_callbacks.js` 中通过 `/eval` 命令直接读取对应字段。

## B. 效果评估（需 ≥ 20 局，主观 + 客观结合）

### B1. 可预测性（v1.1 核心目标）

- [ ] 玩家预判"AI 下一招"的成功率是否下降？（主观评估）
- [ ] 特别关注：有属性克制时，AI 是否偶尔不出克制招？（v1.0 的主要问题）
- [ ] AI 是否出现"明显送分"的随机选择（T 过高时的风险）？

### B2. 换人行为合理性

- [ ] switchP 高时（对手有 counter），AI 使用 UT/VS 的频率是否明显高于 switchP 低时？
- [ ] 岩钉被选中时，是否确实是在对手 bench 岩钉未布的合理时机？
- [ ] "AI 使用 UT/VS → 对手确实换人"的比例是否合理？

### B3. 与 v1.0 的回归对比

- [ ] 自损招式（流星群）使用频率是否与 v1.0 相当（无明显变化）
- [ ] 对手 +2 强化后换人率是否维持 v1.0 水平（改动 A 仍生效）
- [ ] 与 v1.0 相比，AI 整体"出奇制胜"的场景是否增多
