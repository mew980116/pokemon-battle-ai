# 宝可梦战斗AI系统

## 项目作用

本项目是一个宝可梦战斗AI系统，主要用于在宝可梦对战中自动分析战场情况、评估双方精灵属性、预测对手行动，并做出最优的战斗决策。系统能够根据战场实时数据，计算伤害、分析速度、评估交换精灵的利弊，从而帮助玩家获得战斗优势。

## 算法逻辑

### 核心算法

1. **类型克制计算**：通过`typechart`函数实现宝可梦属性之间的克制关系计算，考虑了不同属性之间的伤害倍率。

2. **精灵属性分析**：
   - 分析对手精灵的基础属性、努力值、个体值和性格修正
   - 计算精灵的速度范围，预测先手可能性
   - 评估精灵的物耐和特耐能力

3. **技能效果分析**：
   - 分析技能的伤害输出
   - 评估状态技能的效果和使用时机
   - 考虑天气、场地对技能效果的影响

4. **交换策略**：
   - 基于对手精灵属性和技能，计算最佳交换精灵
   - 考虑联防策略和抗性匹配
   - 评估交换后的进攻和防守能力

5. **决策系统**：
   - 综合考虑伤害输出、生存能力和战场形势
   - 基于概率模型做出最优决策
   - 考虑道具、特性对战斗的影响

- **20201227.js** - 主战斗AI逻辑文件
- **movedata.json** - 技能数据文件
- **db/pokes/weight.txt** - 宝可梦体重数据

## 使用方法

1. 将项目文件中的代码复制到PO的脚本窗口（battle）中
2. 将`movedata.json`复制到PO根目录下
3. 确认脚本中设置的用户ID为你的PO ID
4. 系统会自动检测战斗状态并启动AI
5. AI会根据战场情况自动做出决策，包括攻击、交换精灵等操作

## 技术特点

- **实时分析**：实时分析战场数据，快速做出决策
- **多因素考虑**：综合考虑属性克制、速度、伤害、生存等多个因素
- **自适应策略**：根据对手行为和战场变化调整策略
- **概率模型**：基于概率计算做出最优决策
- **容错机制**：包含错误处理和异常情况的处理

## 注意事项

- 本系统仅适用于特定版本的宝可梦战斗客户端
- 系统依赖于movedata.json和weight.txt数据文件
- 战斗AI的决策基于当前战场信息，可能无法预测对手的隐藏信息

## 代码规范

### 兼容性考虑
- 由于运行环境的JavaScript版本可能较旧，应避免使用ES6+的新特性
- 字符串操作推荐使用传统方法：
  - 使用`indexOf`替代`startsWith`
  - 使用`substring`替代`slice`
  - 使用`for`循环替代`forEach`等数组方法
- 避免使用箭头函数、模板字符串等现代JavaScript特性
- 确保所有变量使用`var`声明，避免使用`let`和`const`

### 输出方法
- 在Qt引擎窗口环境中，使用`print`函数输出信息，而非`console.log`
- 避免使用`setTimeout`等浏览器特有的API
- 直接调用`battle`和`sys`对象，不要重新声明这些变量

### 回调函数格式
- 回调函数应使用`({})`形式，与主战斗逻辑文件保持一致
- 确保所有回调函数参数正确，特别是`spot`、`player`等参数的使用
