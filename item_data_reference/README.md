# 道具数据资料（item_data_reference）

> 存放宝可梦**道具（item）**的原始资料，供整理道具编号对照表、核对 AI 决策逻辑用。
> 与招式资料目录 [move_data_reference/](../move_data_reference/) 平级、同性质。

## 用途

`20201227_v1.2.js` 用整数道具编号判断效果（如专爱系 `[4,5,6]`、剩饭、气球、讲究类、Mega 石 2000-3000、Z 纯晶 3000-4000 等）。代码里这些编号是**裸数字魔法值**，缺人类可读的对照。本目录补上这层资料。

直接相关的待办：TODO「本地可做 1. 字段语义映射」的道具编号对照表（`pokemon.item` 实测 8015 等待解码）。

## 建议包含的资料

- **编号 → 名称对照**：道具整数编号 ↔ 中文/英文名
- **道具效果说明**：伤害修正（讲究头带/眼镜/围巾、生命宝珠）、回复（剩饭/树果）、免疫/规避（气球/红牌）、触发条件
- 编号区间约定（Mega 石 / Z 纯晶 / 树果 等连续段）

## 格式约定

- 原始资料文件命名用小写 + 下划线，ASCII（如 `item_names.txt`、`item_effects.txt`），与 move_data_reference 风格一致。
- 若整理成代码可查的 JSON，放仓库顶层（与 `movedata.json` 平级），不放本目录——本目录只放**原始/参考资料**。
